# コーディング規約

## Python コードスタイル

### 行長の制限

**Pythonコードの行長は79文字を厳守する。このチェックには、自動フォーマッター Black を使用することを必須とし、コミット前に適用すること。**

---

## Black の使用方法

### インストール

```bash
pip install black
```

### 実行方法

```bash
# 単一ファイルのフォーマット
black ファイル名.py

# ディレクトリ全体のフォーマット
black .

# 変更内容の確認のみ(実際には変更しない)
black --check .

# 差分表示
black --diff .
```

### 設定ファイル (pyproject.toml)

プロジェクトルートに `pyproject.toml` を作成し、以下の設定を追加することを推奨:

```toml
[tool.black]
line-length = 79
target-version = ['py38']
include = '\.pyi?$'
```

### コミット前のチェック

コミット前に必ず以下のコマンドを実行してください:

```bash
black --check .
```

エラーが表示された場合は、以下を実行してフォーマットを適用:

```bash
black .
```

## その他のコーディング規約

### セキュリティ

- SQLインジェクション、XSS、認証情報のハードコードなどの脆弱性を含めない
- 外部入力は必ずバリデーションを行う
- パスワードや機密情報は環境変数または設定ファイルで管理

### エラーハンドリング

- 外部I/O(ファイル、ネットワーク、データベース)には必ずtry-exceptブロックを実装
- エラー発生時は、エラーの種類、メッセージ、ファイル名、行番号を出力

### コメント

- 初心者が理解できるよう、詳細なコメントを含める
- 関数やクラスにはdocstringを記述
- 複雑なロジックには説明コメントを追加

### 命名規則

- 変数名・関数名: `snake_case`
- クラス名: `PascalCase`
- 定数: `UPPER_SNAKE_CASE`
- わかりやすく意味のある名前を使用
